技术选择经常会被新工具牵着走，但个人博客其实是一个非常适合“保守选型”的场景。

Jekyll 和 GitHub Pages 的组合有几个很实际的优点：

- 足够稳定
- 生态成熟
- 成本低
- 内容以 Markdown 为中心
- 对长期归档友好

## 它适合什么样的人

如果你更关心以下几点，这套方案会很顺手：

- 想把重心放在写作，而不是折腾站点工程
- 希望博客能长时间稳定托管
- 喜欢内容文件和模板结构都保持清晰
- 希望未来迁移成本尽量低

## 一个很舒服的写作流

Jekyll 的好处之一，是你的文章天然就是仓库里的普通文件。

```md
---
title: "My Post"
date: 2026-04-04 20:30:00 +0800
---

Write in Markdown.
```

这意味着写作、版本管理、回滚、归档、备份都能和日常开发习惯自然接起来。对技术作者来说，这种一致性会带来很强的安心感。
