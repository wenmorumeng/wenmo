## 现在它已经能做什么

这份站点已经从原来的 `WordPress` 方案，改成了适合 `GitHub Pages` 的静态结构。

你现在拥有的是：

- 一个可以直接托管在 GitHub Pages 的博客骨架
- 一套接近 Argon 主题气质的界面
- 一个后续很好维护的 Markdown 写作流程

## 你之后怎么新增内容

在 `docs/_posts` 里新建文章文件，文件名保持 `日期 + 英文标识` 的格式，例如：

```text
2026-04-08-my-new-post.md
```

## 背景图和头像怎么换

先用占位图跑通部署，再逐步替换成你自己的资源会更稳。

你只需要替换：

- `docs/assets/images/avatar.svg`
- `docs/assets/images/banner-day.svg`
- `docs/assets/images/banner-night.svg`

## 这篇文章为什么有目录

文章页面右侧的“文章目录”是前端脚本根据标题自动生成的，所以你只要正常使用 Markdown 标题就行。
